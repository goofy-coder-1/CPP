1. INTRODUCTION TO ENCAPSULATION
--------------------------------
Encapsulation is one of the four foundational pillars of Object-Oriented 
Programming (OOP), alongside Abstraction, Inheritance, and Polymorphism. 

At its core, encapsulation is the mechanism of bundling data (variables) and 
the functions (methods) that operate on that data into a single, cohesive unit 
called a class. Furthermore, encapsulation involves restricting direct access 
to some of an object's components, which is a crucial aspect known as data 
hiding.

Analogy:
Think of a medical capsule. The capsule contains various chemical compounds 
(data and methods) wrapped inside a single protective shell (the class). 
An external entity cannot touch or alter the individual compounds directly 
without going through the capsule shell.


2. THE TWO MAIN PILLARS OF ENCAPSULATION
----------------------------------------
Encapsulation is achieved by combining two complementary concepts:

A. Data Bundling:
   Grouping related variables and functions together under a unified name. 
   Instead of managing separate arrays, integers, and functions across global 
   scope, they are declared together inside a class definition.

B. Data Hiding (Access Specifiers):
   Controlling access to the internal components of the class. C++ provides 
   three primary access specifiers to enforce encapsulation boundaries:
   
   - private: Members declared as private can only be accessed by functions 
     defined inside the same class (or by 'friend' classes/functions). This is 
     the default access level for C++ classes.
     
   - public: Members declared as public can be accessed from anywhere in the 
     program where the object is visible. The public section forms the 
     "interface" of the class.
     
   - protected: Members declared as protected cannot be accessed from outside 
     the class, but they can be accessed by derived (inherited) child classes.


3. IMPLEMENTATION PATTERN: GETTERS AND SETTERS
----------------------------------------------
Because raw data members are typically kept private to prevent direct external 
tampering, classes provide public methods to read and modify that data safely.

- Getter (Accessor): A public member function that returns the value of a 
  private variable. It is usually marked as 'const' to guarantee it does not 
  modify the object's state.
  
- Setter (Mutator): A public member function that assigns a new value to a 
  private variable. Setters are crucial because they allow for validation 
  logic before data is altered.

5. CORE ADVANTAGES OF ENCAPSULATION
-----------------------------------
- Data Protection & Validation: Prevents external code from corrupting the 
  internal state of an object with invalid data (e.g., setting a bank balance 
  to a negative value or an age to 500).
  
- Maintainability and Flexibility: The internal data structures can be changed 
  completely without breaking external code. For example, if you change the 
  'balance' variable from 'double' to an 'int' (representing cents for 
  precision), you only update the getter/setter implementations. The rest of 
  the program calling 'getBalance()' remains untouched.
  
- Code Reusability & Isolation: Encapsulated units are self-contained, 
  making them easy to debug, test, and plug into different parts of a large 
  software application.
  
- Clear Interfaces: Developers using your class don't need to understand its 
  complex internal mechanics. They only need to know what public functions 
  are available.
